# Inducement — Konsep Inti

## Definisi

```
Inducement (IND) = manipulasi titik struktur pasar untuk menciptakan
  narasi palsu mengenai arah harga

IPDA sengaja menciptakan "jebakan" untuk mendorong trader masuk ke posisi
  yang salah, menguras likuiditas mereka sebagai "bahan bakar" untuk
  pergerakan sesungguhnya

Rule inti:
  → Inducement selalu merupakan manipulasi INTERNAL RANGE LIQUIDITY (IRL)
    dalam Trading Range, ATAU penembusan palsu External Range Liquidity (ERL)
  → Agar harga NAIK → harus turun dulu untuk menguras likuiditas di bawah lows
  → Agar harga TURUN → harus naik dulu untuk menguras likuiditas di atas highs
  → Selalu jual di atas highs, selalu beli di bawah lows
```

## Klasifikasi Likuiditas

```
ERL — External Range Liquidity:
  → Selalu berada DI LUAR Trading Range utama
  → Digunakan sebagai Take Profit saat entry pro-tren atau total reversal
  → Old Swing Highs/Lows, Prominent Highs/Lows HTF

IRL — Internal Range Liquidity:
  → Selalu berada DI DALAM Trading Range utama
  → Semua highs/lows yang terbentuk di dalam TR pada TF lebih kecil
  → Bullish TR: semua swing high/low LTF di dalam range
  → Bearish TR: semua swing high/low LTF di dalam range

ARL — Asia Range Liquidity:
  → Special case: ARL bisa berfungsi sebagai IRL maupun IDM
  → Tergantung ukuran range Asia vs struktur HTF
```

# Inducement — Ukuran (Hierarki)

## 3 Ukuran Inducement

```
MAJOR INDUCEMENT (Inducement Utama):
  → Timeframe: W1, D1, H4, H1
  → Semua highs/lows HTF atau zona HTF (liquidity pools)
  → Setiap titik struktur/IRL/ERL yang berusia 1 hari, 1 minggu, atau 1 bulan
  → Asia Range yang bertepatan dengan Daily High/Low = Major IDM
  → Semakin besar IDM → semakin besar pergerakan yang dihasilkan
  → Potensi: swing trade multi-hari

MEDIUM INDUCEMENT (Inducement Sedang):
  → Timeframe: M15, M30, M45, H1
  → Session highs/lows (highs/lows per sesi)
  → Highs/lows berusia kurang dari 1 hari
  → Asia Range (biasanya = Medium IDM)
  → Potensi: pergerakan intraday (ratusan pips)

MINOR INDUCEMENT (Inducement Kecil):
  → Timeframe: M10, M5, M3, M1
  → Setiap titik struktur internal intraday (IRL)
  → Highs/lows dari konsolidasi kecil
  → Asia Range Level (ARL)
  → Potensi: intraday scalp
```

## TF Hierarchy (@DeividFx)

```
W1 mengendalikan D1
D1 mengendalikan H4
H4 mengendalikan H1
H1 mengendalikan M15
M15 mengendalikan M1

Implikasi hierarki:
  → POI M15 TIDAK BISA menghentikan tren H1
  → POI H1 TIDAK BISA menghentikan tren D1
  → Dibutuhkan TF yang LEBIH TINGGI untuk menghentikan tren TF yang lebih rendah

Exception — Minor IDM mengalahkan Major IDM:
  → Terjadi ketika BIAS SMC sejajar satu arah
  → Seluruh order flow bergerak satu arah, menembus setiap level tanpa menghormati hierarki
  → Dalam kasus ini: cari entry saat Medium IDM diinduksi, TP di Major IDM

SMT (Smart Money Trap) via Hierarki:
  → Ketika LTF membentuk ChoCh/IFC ekstrem di zona HOB H1/H4
  → Ekstrem LTF tersebut menjadi SMT / Inducement Point
  → Contoh: ChoCh bearish M5 tepat di OB M30 → extremum M5 = SMT
  → SMT bisa diperdagangkan hanya jika M15 atau lebih tinggi (untuk RR yang wajar)
  → Tapi harus sadar bahwa setelah reaksi SMT, harga AKAN berbalik untuk ambil likuiditas yang dihasilkan
```

# Jenis-Jenis Inducement

## Tipe 1 — False BOS (Penembusan Struktur Palsu)

```
Definisi:
  → Manipulation yang mendorong sebagian besar trader untuk melihat TREN BARU
  → Digunakan untuk perdagangan LANJUTAN (continuation)
  → Apa yang trader lihat sebagai BOS bullish = sebenarnya clearing likuiditas bearish

Bullish scenario (struktur HTF bullish):
  → Harga membuat lower low (terlihat bearish di LTF)
  → Trader ritel masuk SELL mengikuti "breakdown"
  → Aktual: harga clearing likuiditas di bawah low HTF
  → Setelah likuiditas terambil → harga naik tajam (kelanjutan bullish)

Bearish scenario (struktur HTF bearish):
  → Harga membuat higher high (terlihat bullish di LTF)
  → Trader ritel masuk BUY mengikuti "breakout"
  → Aktual: harga clearing likuiditas di atas high HTF
  → Setelah likuiditas terambil → harga turun tajam (kelanjutan bearish)

Kunci penghindaran:
  → Selalu identifikasi Strong Points vs Weak Points
  → Lihat TR HTF sebelum mengkonfirmasi BOS apapun
  → Yang kelihatan BOS di LTF = kemungkinan besar IDM dalam konteks HTF
```

## Tipe 2 — False LTF Continuation (Kelanjutan LTF Palsu)

```
Definisi:
  → Terjadi ketika trader terlalu fokus pada LTF dan mengabaikan HTF
  → Mendorong trader percaya bahwa tren LTF akan berlanjut
  → Aktual: itu hanya koreksi dalam tren HTF yang lebih besar

Bearish tipe 2 (struktur HTF bullish):
  → Di LTF: terlihat struktur bearish yang valid (HH-HL breakdown menjadi LL-LH)
  → Trader LTF masuk sell dengan konfirmasi LTF penuh
  → Di HTF: ini hanya retracement dalam tren bullish
  → Setelah mencapai POI bullish HTF → harga naik kuat (kelanjutan HTF)
  → Semua entry sell LTF terhapus

Bullish tipe 2 (struktur HTF bearish):
  → Di LTF: terlihat struktur bullish yang valid
  → Trader LTF masuk buy dengan konfirmasi penuh
  → Di HTF: ini hanya koreksi dalam tren bearish
  → Setelah mencapai POI bearish HTF → harga turun kuat

Rule: dalam struktur bullish HTF, BOLEH terus menjual menggunakan struktur LTF
  SAMPAI harga mencapai POI bullish HTF
  → Setelah itu → switch ke continuation bullish
```

# Formula Inducement (Vector FX)

## Inducement Amount Formula

```
Inducement Amount Taken = Distance of Movement (dari OB mitigation) × Momentum

Momentum bearish kuat:
  → Sedikit inducement dibutuhkan untuk membuat sell move medium/large
  → Price membutuhkan "bahan bakar" lebih sedikit

Momentum bullish kuat:
  → Banyak inducement dibutuhkan untuk membuat sell move medium/large
  → Price membutuhkan "bahan bakar" lebih banyak untuk melawan momentum

Aplikasi:
  1. Evaluasi momentum HTF terlebih dahulu
  2. Estimasi berapa banyak inducement yang dibutuhkan sebelum real move
  3. Momentum membantu identifikasi HTF influence dan target price
  4. Partial profit atau SL di BE karena kita tidak pernah tahu pasti
```

# LIT Cycle — Siklus Induksi dan Likuiditas

## 3 Fase LIT Cycle

```
FASE 1 — LIQUIDITY FORMATION (Pembentukan Likuiditas):
  → Harga menciptakan bentuk likuiditas SEBELUM pergerakan besar berikutnya
  → Tujuan: menjebak trader masuk ke pasar sebelum pergerakan sesungguhnya
  → Bisa muncul sebagai: support, resistance, trendlines, konsolidasi, Asia Range
  → Bisa terkumulasi ke satu arah (bullish/bearish channel)
  → KAPAN: terjadi SEBELUM sesi volatilitas tinggi (London/NY)
             biasanya selama Asia (volatilitas rendah)
             untuk kemudian di-induce saat Frankfurt/London/NY dibuka

FASE 2 — INDUCEMENT (Induksi):
  → Terjadi SETELAH Pembentukan Likuiditas
  → Price memanipulasi beberapa highs/lows dekat POI kita
  → Biasanya: pergerakan cepat, sedikit candle, atau hanya wick
  → Sering terjadi saat news fundamental dirilis
  → Terjadi dalam Killzones (waktu spesifik)
  → Menciptakan "day trap" → Stop Hunt
  → Semakin besar IDM → semakin besar pergerakan selanjutnya

FASE 3 — MITIGATION atau EFFICIENCY (Mitigasi/Efisiensi):
  → Price mengurangi POI atau mengisi ketidakefisienan (FVG)
  → Mitigasi terjadi di Decisional POI (prob. sedang) dan Extreme POI (prob. tertinggi)
  → Mitigasi di Decisional sering digunakan sebagai "bahan bakar" likuiditas tambahan
    agar saat mencapai Extreme, pergerakan menjadi lebih besar
  → POI HTF = paling reliable
  → Sebagian besar mitigasi terjadi selama Killzone
  → SMT dapat muncul DALAM siklus ini (saat fase Inducement + Mitigation bersamaan)
```

# Session Timing (EST)

## Jadwal Sesi

```
Sydney Open:      16:00 - 18:00 EST (pembukaan saja)
  → "Mereset" algoritma, candle kecil, volatilitas sangat rendah
  → AUD/NZD bisa bergerak; tidak digunakan untuk trading

Asia Session:     17:00 - 02:00 EST
  → FASE 1: Pembentukan Likuiditas
  → Konsolidasi → menciptakan likuiditas untuk London/NY
  → Memahami anatomi Asia = kunci entry terbaik di sesi volatil berikutnya

Frankfurt Open:   02:00 - 03:00 EST (satu jam sebelum London)
  → Sering bergerak BERLAWANAN arah yang akan diambil London
  → Dua tujuan:
    a. Menciptakan likuiditas → London reversal
    b. Melakukan Inducement + Mitigation → London continuation

London Session:   03:00 - 11:00 EST  |  London KZ: 03:00 - 05:00 EST
  → Sesi volatil pertama
  → Skenario bergantung pada variasi Daily Cycle
  → Bisa: reversal dari Frankfurt; atau continuation dari Frankfurt
  → Favorite scenario: berdasarkan apa yang terjadi di Asia + Frankfurt

New York Session: 08:00 - 16:00 EST  |  NY KZ: 08:00 - 11:00 EST
  → Sesi volatil kedua
  → Bisa: reversal dari London; atau continuation dari London
  → London+NY overlap = volatilitas terbesar = injeksi likuiditas terbesar
  → "Magic Minutes" (MMM): 1,5 jam setelah setiap pembukaan sesi

Summary timing sequence:
  Asia (liquid. formation) → Frankfurt (IDM/false run) → London (reversal/continuation)
                           → New York (continuation/reversal dari London)
```

# Asia Range Anatomy

## Komponen Asia Range

```
ASIA HIGH:
  → Titik tertinggi yang dicapai harga selama sesi Asia
  → Hampir selalu dimanipulasi (di-sweep) sebelum pergerakan sesungguhnya
  → Exception: tergantung variasi Daily Cycle yang terjadi

ASIA LOW:
  → Titik terendah yang dicapai harga selama sesi Asia
  → Hampir selalu dimanipulasi (di-sweep) sebelum pergerakan sesungguhnya

ASIA MIDLINE (50% level):
  → Rata-rata rentang Asia (bukan sekadar garis)
  → Zona kecil di mana harga sering kembali untuk mereduksi POI relevan
  → Kunci untuk variasi "Asia Sweep + Midline"

LIQUIDITY DALAM RANGE ASIA:
  → Bisa berbentuk trendlines, channels kecil, konsolidasi internal
  → Dikombinasikan dengan pergerakan Frankfurt → creates additional IDM
  → Asia range pip height: ≥40 pips = sering OB lanjutan untuk London;
                           <40 pips = accumulation / peningkatan likuiditas satu sisi
```

# 6 Variasi Daily Cycle

## Overview

```
Daily Cycle = pola berulang yang menggambarkan bagaimana IPDA bergerak setiap hari
  → Selalu terdiri dari komponen: Asia Range, Frankfurt, London, New York
  → Fokus utama: Forex
  → Variasi menentukan KAPAN dan BAGAIMANA masuk
```

## Variasi 1 — Normal Day Liquidity

```
Pattern: Asia konsolidasi → Frankfurt sweep SATU sisi → London continuation

Bullish:
  → Asia membentuk range (likuiditas terbentuk)
  → Frankfurt sweep Asia Low (IDM minor)
  → London open: continuation naik dari POI bullish
  → NY: kelanjutan atau pullback ke POI baru

Bearish:
  → Asia membentuk range
  → Frankfurt sweep Asia High (IDM minor)
  → London open: continuation turun dari POI bearish
  → NY: kelanjutan atau pullback

Kunci: satu sisi range Asia di-sweep, sisi lain menjadi target
```

## Variasi 2 — Asia Whipsaw (AWS)

```
Pattern: Asia sweep KEDUA sisi (both high AND low) → London/NY continuation

Bullish:
  → Asia atau Frankfurt sweep Asia Low dulu
  → Kemudian sweep Asia High juga
  → London: entry dari POI setelah whipsaw selesai, direction ditentukan HTF
  → Kunci: bias HTF menentukan arah continuation setelah whipsaw

Bearish:
  → Sebaliknya dari bullish

Karakteristik:
  → Asia pip height kecil → kandidat utama whipsaw
  → Setelah whipsaw: baru ada arah yang jelas
  → Trading: tunggu whipsaw selesai, entry London KZ
```

## Variasi 3 — Asia Sweep + Midline

```
Pattern: sweep SATU sisi Asia → harga kembali ke MIDLINE (50%) untuk POI entry

Bullish:
  → Frankfurt/London awal sweep Asia High
  → Harga tetap di atas Asia High
  → Kembali ke 50% Asia Range (Midline) untuk mitigasi POI
  → Entry di POI Midline selama London KZ
  → Direction: pro-trend bullish

Bearish:
  → Frankfurt/London awal sweep Asia Low
  → Harga tetap di bawah Asia Low
  → Kembali ke 50% Asia Range (Midline) untuk mitigasi POI
  → Entry di POI Midline selama London KZ

Key: harga tidak menembus seluruh sisi berlawanan;
     hanya kembali sampai 50% range untuk mitigasi
```

## Variasi 4 — Asia Sweep + Midline Premium/Discount

```
Pattern: mirip variasi 3, tapi harga kembali melewati Midline ke Premium/Discount

Bullish:
  → Sweep Asia High → harga tetap di atas
  → Kembali ke DISCOUNT Asia Range (di bawah 50%) untuk POI
  → Entry di POI discount selama London

Bearish:
  → Sweep Asia Low → harga tetap di bawah
  → Kembali ke PREMIUM Asia Range (di atas 50%) untuk POI
  → Entry di POI premium selama London

Pilihan variasi 3 vs 4:
  → Lihat kualitas POI yang tersedia
  → Pilih POI yang lebih jelas/confirmed dari sisi Order Flow
  → Midline POI lebih jelas → variasi 3; Premium/Discount POI lebih jelas → variasi 4
```

## Variasi 5 — Mitigation Asia High/Low

```
Pattern: sweep satu sisi → harga kembali ke EKSTREM range (High atau Low) untuk mitigasi

Bullish:
  → Frankfurt sweep Asia High → harga tetap di atas
  → Kembali ke Asia High level (bukan 50%, bukan Discount; langsung ke top)
  → Mitigasi POI yang sangat jelas tepat di Asia High level
  → Entry selama London KZ atau NY KZ

Bearish:
  → Frankfurt sweep Asia Low → harga tetap di bawah
  → Kembali ke Asia Low level untuk mitigasi POI
  → Entry selama London atau NY

Catatan:
  → Variasi yang lebih jarang terjadi
  → Hanya valid jika struktur intraday sejalan dengan direction
  → Semua faktor (struktur, POI, session timing) harus sejajar
```

## Variasi 6 — No Asia Sweep (Tidak Ada Sweep Range)

```
Pattern: Frankfurt tidak mampu sweep satupun sisi Asia → POI dalam range untuk London

Bullish:
  → Volatilitas Frankfurt sangat rendah; tidak sweep High atau Low
  → Jika di dalam range ada POI bullish dengan probabilitas tinggi
  → London open → langsung entry di POI tersebut
  → Valid karena London baru saja dibuka (momentum tinggi)

Bearish:
  → Sama tapi POI bearish dalam range
  → London open → entry di POI tersebut

Kunci:
  → POI harus sangat jelas dan high-probability
  → Frankfurt tidak bergerak = likuiditas terakumulasi → London explosive
```

# 2-Phase Inducement Theorem (Vector FX)

## Konsep Inti

```
Tujuan: menemukan 2 fase inducement untuk memvalidasi REVERSAL
         sebelum memperdagangkan continuation

Definisi:
  Inducement = jebakan likuiditas fraktal yang terjadi di atas/bawah
               Valid Orderblock untuk memicu pergerakan besar

Setelah 2-Phase Reversal selesai → ikuti order flow dengan continuation
  (IPB = Internal Price Basing / pullback) sampai mencapai target Fase 1 Reversal

Fraktalitas:
  → Berlaku untuk semua TF (Daily-4H, 4H-1H, 1H-15m/5m, 15m/5m-1m)
  → Untuk trade daily, lihat 2 fase terjadi di 5m/15m level ke atas
  → Jika 2 fase terjadi di 1H/4H/Daily → tunggu hingga TF ≥15m untuk entry

Aturan khusus:
  → Jika Inducement Fase kecil dalam TF referensi → dua Fase 2 kecil bisa terbentuk
```

## Fase 1 — Mitigation-Inducement

```
Definisi:
  → Titik akhir dari penjual/pembeli AWAL sebelum Orderblock utama
  → Price memitigasi penjual/pembeli awal ini untuk MENGUMPULKAN lebih banyak
    orders yang kemudian akan dilikuidasi
  → SL penjual/pembeli awal berada DI BAWAH zona sebenarnya → dikeluarkan
    saat Orderblock dimitigasi sebagai inducement

Mekanisme:
  → Harga membangun "rantai Supply/Demand palsu" di bawah/atas OB sebenarnya
  → Semua jenis trader bisa terinduksi di sini (bahkan SMC trader)
  → Mitigasi Fase 1 = harga bereaksi pada level DI BAWAH Imbalance
    tanpa menggunakan zona refinement yang sebenarnya

Take Profit Fase 1:
  → TP terbaik = bagian ATAS level bujukan-mitigasi (jika membeli)
  → TP terbaik = bagian BAWAH level bujukan-mitigasi (jika menjual)
```

## Fase 2 — ChoCh Inducement

```
Definisi:
  → Lapisan manipulasi KEDUA sebelum atau sesudah tap-in OB asli
  → Bentuk inducement terakhir yang dibutuhkan sebelum entry trade

4 Sub-tipe Fase 2:

1. ChoCh Inducement [Paling Pasif]:
   → Terjadi SETELAH memanfaatkan refinement OB
   → Mengurangi/melanggar Vector Level → membuat OB valid di M1 sebagai konfirmasi
   → Entry: tunggu mitigasi zona yang dibentuk ChoCh

2. Liquidity Rest Before Refined OB / Smart Money Trap [Semi-Agresif]:
   → Kumpulan orders di bawah/atas OB yang GAGAL menerima orders besar
   → Bentuk likuiditas yang memanipulasi trader SEBELUM mencapai zona sebenarnya
   → Jika akumulasi pra-tap-in besar → lebih sedikit konfirmasi dibutuhkan

3. Two Phase 1 / One Long Phase 1 [Agresif]:
   → Dua mitigasi inducement ATAU satu mitigasi panjang dengan Imbalance
   → Terjadi sebelum OB yang jelas
   → Sering terlihat selama pergerakan bullish/bearish yang kuat

4. Second Failure Refinement [Agresif]:
   → Setelah Fase 1, harga kembali miss refinement secara impulsif
   → Break internal structure → kemudian di-sweep sebagai IDM kedua
   → Kemudian terjadi aksi dari zona sebenarnya

Note: Fase 2 SEBELUM OB = entry lebih agresif (Limit Order dari zona)
      Fase 2 SETELAH OB = entry lebih pasif (konfirmasi M1 setelah tap-in)
```

## 4 Setup Types (Vector FX)

```
REVERSAL SETUPS — Butuh 2 Fase Inducement:
  1. Bullish Reversal (Setup 1):
     → 2 fase IDM selesai → entry dari OB demand → target HTF supply/ERL

  2. Bearish Reversal (Setup 1):
     → 2 fase IDM selesai → entry dari OB supply → target HTF demand/ERL

CONTINUATION SETUPS — Hanya 1 Fase (TF Exception):
  3. Bullish Continuation/IPB (Setup 2):
     → Setelah reversal valid → Fase 1 IDM saja (TF lebih rendah)
     → Entry pullback → continuation ke target

  4. Bearish Continuation/IPB (Setup 2):
     → Setelah reversal valid → Fase 1 IDM saja
     → Entry rally → continuation turun ke target
```

# 4 Counterparts of a Setup (Vector FX)

## Komponen Setup

```
1. ORDERBLOCK:
   → Identifikasi level OB dengan probabilitas tinggi
   → OB valid: dekat dengan posisi IDM (price mitigasi sesuatu proximal ke IDM)
   → Tidak selalu harus candle terakhir sebelum impulsive move

2. INDUCEMENT:
   → Jebakan yang diciptakan untuk menangkap massa (think like a bank!)
   → Harus ada inducement di atas/bawah OB untuk memvalidasi entry
   → Tanpa inducement → probabilitas rendah

3. BUILD-UP LEG:
   → Price action yang terbentuk SEBELUM mitigasi OB dan sweep IDM
   → Bisa berupa: continuational structure atau weak build-up structure
   → Vector Level menentukan arah pergerakan berikutnya

4. TARGETS:
   → Rule: trade sampai next potential reversal-inducement level dari active zone
   → Klasifikasi IDM menentukan kekuatan target:
     Minor IDM → pullback saja (hold through)
     Medium IDM → pullback saja (hold through)
     Major IDM → potential reversal (partial di sini)
   → Partial di levels of uncertainty karena tidak selalu benar 100%
```

## 4 Secondary Elements

```
1. TIME × PRICE:
   → Price hanya memanipulasi di waktu-waktu tertentu
   → London Open (+30m), NY Open (+30m), MMM times (Magic Minutes)

2. MOMENTUM:
   → Terbentuk dari sifat fraktal HTF-LTF
   → Membantu identifikasi: targets, IDM needed, HTF influence, shifts (reversals)

3. INDUCEMENT STRENGTH:
   → Minor, Medium, Major → menentukan berapa banyak IDM yang dibutuhkan

4. DAILY CYCLE FACTORS:
   → Mana dari 6 variasi Daily Cycle yang sedang terjadi
   → Menentukan skenario yang paling mungkin di setiap sesi
```

## Trend Run Continuation (Arah Sangat Kuat)

```
Bearish (Trend Run Down Continuation):
  Tidak ada demand yang bisa memecahkan OB pro-tren utama
  1. Frankfurt: ambil IDM Fase 1 (Asia high), stop demand → MSS M5
  2. Sesi Frankfurt (Pre-London): ciptakan IDM Fase 2 untuk London Open
  3. London Open: mitigasi supply zone Frankfurt {Sell 1}
  4. London Magic Minutes (+1.5h): mitigasi London Open, sweep liquidity {Sell 2}
  5. NY Open: mitigasi supply chain; NY Magic Minutes → final intraday push down {Sell 3}

Bullish (Trend Run Up Continuation):
  Tidak ada supply yang bisa memecahkan OB pro-tren utama
  [Mirror dari bearish]
  1. Frankfurt: sweep Asia Low (IDM Fase 1), stop supply → MSS M5
  2. London Open: mitigasi supply Frankfurt, sweep Fase 1 IDM → ChoCh impulsif
  3. London Magic Minutes: mitigasi London Open zone {Buy 2}
  4. NY: lanjutkan demand chain, NY Magic Minutes → final intraday push up {Buy 3}
```

## Slightly Bearish/Bullish (Medium Momentum)

```
Slightly Bearish (Arah Sedikit Bearish - Medium Momentum):
  Ada demand berlawanan medium/kuat yang melawan tren turun
  Kurang momentum → bisa menandakan likuiditas korektif untuk reversal bullish nanti
  1. Frankfurt: sweep Asia Low → mitigasi zone demand pullback → ChoCh naik
  2. London Open: mitigasi Frankfurt, tinggalkan OB valid dengan IDM ChoCh Fase 2
  3. London Magic Minutes: mitigasi hambatan → naik ke Asia High [Buy 1]
```

# Orderblock — Identifikasi & Validasi (Vector FX)

## 5 Tipe Orderblock

```
1. ENGULFED OB (OB yang Ditelan):
   → Candle buy/sell sebelum pergerakan turun/naik besar
   → Ada IDM yang jelas di atas/bawahnya
   → Probabilitas tertinggi karena institusi transfer uang dan clear imbalance
   → Tanda: candle tertelan sepenuhnya oleh impuls berikutnya

2. IMBALANCE OB:
   → Ada V-SR besar berbentuk V tanpa zona bersih dan Imbalance
   → Zona valid = setelah sweep atas/bawah IDM Fase 1 + sapuan candle terakhir
   → Tujuan: mitigasi V-SR
   → IDM tidak boleh mendekati extremum; hanya mendekati dasar/atas area kegagalan
   → Cari zona LTF dalam Imbalance

3. WICK/HAMMER OB:
   → 50% wick dari IFC tidak tanggung-tanggung
   → Digunakan sebagai OB terutama jika masih ada likuiditas di bawah/atasnya
   → Probabilitas tinggi: menunjukkan institusi "memukul" zona tersebut lalu reversal

4. COMPLEX OB (Smart Money Trap):
   → Clean OB di atas/bawah V-SR (yang ciptakan Imbalance) TAPI tanpa IDM
   → Harga menjalankan blok ini lalu melonjak (swept sebagai Smart Money Trap)
   → Tandai dengan kotak di atas blok jebakan

5. FASE 1 INDUCEMENT SWEEP REACTION:
   → Harga tidak menggunakan Clean OB → berbalik tepat setelah IDM Fase 1
   → Institusi: "selesai" karena sudah clear buy-side/sell-side liquidity chain
   → Cari OB setelah sweep LTF; atau react langsung ke Imbalance
   → Mengilustrasikan ide utama liquidity-inducement trap trading
```

## 3 Aturan Validasi OB

```
Untuk OB valid dan dapat diperdagangkan:
  1. IMBALANCE: harus menciptakan inefisiensi karena sifat impulsif (temukan di M1)
  2. IMPULSIF: harus menciptakan atau efisien memitigasi High-Volume order flow
               (punya alasan di market)
  3. MENCIPTAKAN IDM: di atas/bawahnya harus ada IDM-mitigasi Fase 1
                      dalam TF referensi
                      (bisa Fase 2 sebelumnya atau dibuat setelah mitigasi OB)

OB Tips:
  → OB yang sweep likuiditas = probabilitas lebih tinggi
  → Refinement ke M1 dengan overlay Supply/Demand box; jangan over-refine
  → Jika harga sedikit masuk OB saat refinement = mitigasi gagal → akan di-sweep
  → Hierarchy refinement: Daily OB → 4H; 4H OB → 15m; 15m OB → 5m/1m; 5m OB → 1m
```

# Trap Trading (@DeividFx)

## Filosofi

```
Trap Trader = beroperasi MELAWAN trader ritel

Trader ritel:
  → Masuk saat melihat pola chartist (Double Top/Bottom, Trendline, S/R, HCH)
  → Lokasi entry dan SL mereka sangat PREDICTABLE
  → SL mereka = target kita (likuiditas untuk diambil)

Cara kita:
  → Identifikasi entry lokasi trader ritel
  → Masuk BERLAWANAN arah mereka saat mereka terperangkap
  → TP = exit aman di mana ditemukan likuiditas ritel (1:4 RR minimum)
  → Tidak perlu mengejar 1000 pip; ambil di tempat yang tepat dan pergi
```

## 4 Pola Ritel yang Digunakan

```
1. DOUBLE TOP / DOUBLE BOTTOM:
   → Retail masuk sell di Double Top (SL di atas)
   → Kita: identifikasi Double Top sebagai IDM + OB demand di bawah
   → Entry buy setelah sweep retail sell-side SL → continuation naik
   → (Sebaliknya untuk Double Bottom)

2. FAKE BREAKOUT (Bullish/Bearish):
   → Retail masuk continuation saat breakout dari range
   → Kita: identifikasi breakout sebagai IDM/SMT
   → Entry berlawanan setelah breakout palsu + konfirmasi
   → Target: kembali ke dalam range ke level sebaliknya

3. HEAD AND SHOULDERS (HCH / Shoulder-Head-Shoulder):
   → Retail masuk sell setelah "neckline" break
   → Kita: identifikasi neckline break sebagai IDM
   → Entry buy dari demand di bawah "neckline" → reversal
   → Target: SL retail yang berada di atas right shoulder

4. RANGE BREAKOUT / S&R:
   → Retail masuk continuation dari range breakout
   → Kita: identifikasi breakout sebagai IDM + supply di atas range
   → Entry sell setelah false breakout → back into range
   → Target: likuiditas yang dihasilkan retail di dalam range
```

# Validasi POI dengan Likuiditas (@DeividFx)

## Menemukan POI Valid

```
Prinsip:
  → Setiap maximum/minimum yang TIDAK menyentuh POI dan price terus bergerak
    → POI tersebut mengandung likuiditas sangat tinggi
    → Berubah menjadi zona kuat untuk entry yang baik

Validasi via EQL/EQH:
  → 2+ minimum dekat POI tanpa menyentuh = EQL
  → Institusi ingin tarik semua SL sebelum ambil posisi ekstrem
  → Entry: setelah EQL/EQH di-sweep + konfirmasi OB di zona

Validasi via Trendline Liquidity:
  → 2+ maximum/minimum dekat POI membentuk trendline
  → Retail sell di trendline = SL di atas → likuiditas tinggi
  → Price harus sweep semua SL tersebut SEBELUM bergerak ke arah yang benar

PREMIUM/DISCOUNT validation:
  → Ukur dengan Fibonacci untuk menentukan 50% impuls
  → POI buy harus berada di DISCOUNT (di bawah 50%) dari fraktal
  → POI sell harus berada di PREMIUM (di atas 50%) dari fraktal
  → Kombinasikan dengan IDM yang terbentuk sebelum POI
```
