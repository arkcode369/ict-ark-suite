# Divisions of Time — Candle Structure Logic

## Dua Teori Pembagian Candle

```
3-Part Division:
  Gunakan untuk: Daily, Weekly candle
  Open → 1/3 → 2/3 → Close

4-Part Division:
  Gunakan untuk: Monthly candle
  Open → 1/4 → 2/4 (midpoint) → 3/4 → Close

Fixed values per candle:
  O (Open)   = Fixed Time
  C (Close)  = Fixed Time
  Midpoint   = Fixed Time = (O+C)/2

Moving values:
  H (High)   = not fixed
  L (Low)    = not fixed
```

> "You're trading a Higher Timeframe Candle with this premise in mind."

## HOD/LOD hingga Midpoint Time

```
LOD/HOD terbentuk sampai midpoint waktu candle:
  "The HOD/LOD until New York"
  "The High of the Week until Thursday/Wednesday"
  "The Low of the Day until 9:00"

Interpretasi:
  → Jika bullish: LOD terbentuk sebelum midpoint (London/NY open)
  → Jika HOD/LOD belum terbentuk saat midpoint → expect extension

Market menghabiskan 70% waktu dalam Accumulation.
Jangan expect Expansion ketika market sedang Accumulate.
```

## Monday's Opening Price

```
Once Monday's Opening Price is in:
  → 50% of the weekly puzzle is solved
  → Goal = participate in Expansion of the HTF

OSOK (Open, Seek, Once Confirmed):
  → Setelah Monday's OP diketahui → frame weekly range

Setiap Buy dalam kondisi Bullish harus DI BAWAH Opening Price
Setiap Sell dalam kondisi Bearish harus DI ATAS Opening Price
```

# Opening Price Framework

## Bullish vs Bearish Relative to OP

```
BEARISH:
  Range ABOVE Opening Price → CSD → Move Lower
  OBs, BKRs, OTE, FVGs terbentuk DI ATAS OP

BULLISH:
  Range BELOW Opening Price → CSD → Rally
  OBs, BKRs, OTE, FVGs terbentuk DI BAWAH OP
```

> All PD Arrays (OB, BKR, OTE, FVG) form below OP when Bullish, and above OP when Bearish.

## Change in State of Delivery (CSD)

```
Konfirmasi CSD sebelum entry:
  Bullish: CSD = close above a bearish candle / break of swing high
  Bearish: CSD = close below a bullish candle / break of swing low

"Watch if it closed above/below or falsely closed. It's super important."

Refusal signals:
  Refusal to close BELOW swing low that broke a High → bullish continuation
  Refusal to close ABOVE swing high that broke a Low → bearish continuation
```

# Power of 3 — Waktu & Sesi

## Hierarchy Waktu (Monthly → Weekly → Daily → Intraday)

```
Monthly PO3:
  Week 1  = Accumulation (skip/wait)
  Week 2  = Manipulation (high/low of month forms here)
  Week 3  = Distribution / Continuation → look for entries
  Week 4  = Chopfest (skip, unless ranging Monthly expected)

Weekly PO3:
  Monday   = Accumulation → let it play out, skip trading
  Tuesday  = Manipulation → High/Low of Week forms
  Wed-Thu  = Distribution → ride to target
  Friday   = Return to Range (TGIF)

Daily PO3:
  00:00 – 09:30 = Accumulation
  09:00 – 09:30 = Accumulation (Judas starts at 09:30, NOT 09:00)
  09:30 = Judas Swing begins
  Midpoint of day = HOD/LOD expected to be in

Intraday:
  "The Judas comes at 2:00 not 00:00"
  "We skip the first half of the hour"
  "9:30 not 9:00"
```

## Timing Confirmasi Turtle Soup

```
Konfirmasi Turtle Soup Low/High:
  1. Wait for confirmation of a High being broken
     (Opening Price High, High that broke a Low)
  2. Wait for sections of the candle to play out
     → Entry at section 2 or 3, ride to High of candle (section 1)

SL placement berdasarkan timing:
  Week 2 High/Low
  Tuesday High/Low
  London High/Low
  → "You're trusting this Low formed during that certain time to hold"
```

## LOD/HOD Rule

```
LOD/HOD = 50% point of duration of the candle:
  "I'm going to catch the High of the Week till Thursday/Wednesday"
  "I'm going to catch the Low of the Day till 9:00"

LOD = awal = entry BUY hingga 50% durasi
HOD = awal = entry SELL hingga 50% durasi
```

# Candle Tipe & AMD

## 4 Daily Templates

```
Hanya ada 4 cara daily candle bisa close:
  1. Classic Buy  = O-L-H-C (bullish: Low dulu, High akhir)
  2. Classic Sell = O-H-L-C (bearish: High dulu, Low akhir)
  3. Seek & Destroy Neutral = Close = Open (doji-like)
  4. Seek & Destroy Bullish/Bearish = Close ≠ Open tapi kedua sisi diambil

Sequence: Expansion (1&2) → Consolidation (3&4) → Retracement/Reversal (5&6)
```

## Candle Engulf Logic

```
Bullish market: Bearish candle akan diengulf oleh candle berikutnya
Bearish market: Bullish candle akan diengulf oleh candle berikutnya

Watch: "If it closed above/below or falsely closed"
  Close above = higher prices
  Close below = lower prices
  Wick above (tidak close) = bearish reversal signal
  Wick below (tidak close) = bullish reversal signal
```

## True Breakout vs False Breakout (Turtle Soup)

```
TRUE BREAKOUT (Expansion) — tandanya:
  1. 3 candle consecutive close menembus level
  2. FVG tersisa di belakang setelah breakout
  3. Instant rejection dari FVG kemudian expansion
  4. Acceleration away from level instantly
  5. Opposing liquidity sudah diambil terlebih dahulu

FALSE BREAKOUT / Turtle Soup — tandanya:
  1. Harga menolak close di atas/bawah level
  2. Tidak ada FVG tersisa setelah "breakout"
  3. Tidak ada acceleration away
  4. Harga kembali ke dalam range dengan cepat
  5. "The first breakout of a range is almost always the false one"

Warning signs FALSE TS (= it's a real breakout):
  → Opposing liquidity taken
  → Close through level with 2 candles
  → Close through with 3 candles leaving FVG behind
  → Acceleration away from level instantly from FVG
```

## 3 Reaksi Harga di Key Level

```
1. V-shaped reversal = instant reaction
   → Biasanya terjadi pada: Orderblock
2. M-shaped reversal = failure swings + Turtle Soup
   → Biasanya terjadi pada: Turtle Soup x MSS
3. Multiple taps = least favorable
   → Kurang andal, lebih banyak noise

Tips:
  Anticipate MANA tipe reaksi yang akan terjadi = hanya bisa dari backtesting
  "If the reaction is not instantaneous = no good"
```

# Rejection Block

## Definisi & Cara Trading

```
Rejection Block:
  = Orderblock dengan Long Wick (wick panjang)
  = RB di HTF = OB pada LTF (fractal)

Cara trade RB (sama seperti SMT):
  1. Tunggu CSD di LTF: close above bearish candle
  2. Sambil violating Bearish PD Array (Swing High)

Candle di sebelah Rejection Block biasanya akan diambil.

"If they close above the rejection block → going to 50% of wick"
"If they close above 50% of wick → going to end of wick"
```

# Range Trading — Turtle Soup Framework

## Aturan Dasar Range

```
Rule #1: Buy Low, Sell High
  → Don't Buy High
  → Don't Sell Low

Posisi relatif Opening Price:
  Every Buy when Bullish = below Opening Price
  Every Sell when Bearish = above Opening Price

Range trading rules:
  Buy Range Low
  Sell Range High
  If not sure:
    Never Buy Range High
    Never Sell Range Low

Never deal with price at 50% of Range (terlalu choppy)
  → 50% = bisa jadi S/R dari kedua arah
  → Expect Stop Run at 50% of Range
```

## Range Purge Logic

```
Ketika satu sisi range diambil:
  Target 1: 50% of the range
  Target 2: Range High/Low (opposing end)
  Target 3: Swing High/Low yang menyebabkan purge

"A Purge on one side will lead us to the other side"

Model: Generating Liquidity → Range Low Purged → Buyside intact
  → Reasonably anticipate: 50% dan Range High

Blend dengan:
  HTF PD Array + Time + Market Profiles
```

## Breakout Trading (Kapan Boleh)

```
Satu-satunya waktu boleh menjadi breakout trader:
  → Ketika mereka create a PD Array at Range High/Low

Setelah false breakout:
  → Never Long atau Short dari posisi itu lagi
  → "We have shifted. We are now trading Bullish to Bearish (or vice versa)"

Sequence:
  Consolidation → Turtle Soup (false) → Expansion (real)
```

# Turtle Soup — Detail & Entry

## Jenis-Jenis Turtle Soup

```
1. Range TS     = false breakout dari defined range
2. AMD TS       = false breakout dalam AMD cycle (saat manipulation)
3. High/Low Test TS = simple sweep dari significant high/low

Dua perspektif:
  Against the trend: false breakout yang reversal
  With the trend: breakdown/breakout yang continuation
  → "Keduanya identik — hanya berbeda di timing dan durasi hold"
```

## Kapan TS Probabilitas Tertinggi

```
Turtle Soup terjadi di:
  1. London Open
  2. London Killzone ending (paling baik)
  3. NY Open
  4. London Close
  5. End of 3pm Macro

Konteks tambahan:
  1. PWH, PDH (previous week/day high)
  2. 3-drives formation (double top palsu + 3rd drive)
  3. Dalam Killzone
  4. Di Distribution phase AMD cycle

50% of any Price Leg = good place for Turtle Soup
50% of any Range = expect continuation Turtle Soup
```

## Entry Techniques Turtle Soup

```
3 Entry Techniques:
  1. HTF PDA dengan SL di HTF PDA
  2. ITL/ITH sebelumnya (to the left) sebagai SL
  3. LTF CSD (Change in State of Delivery)

Entry hierarchy:
  Entry 1: Turtle Soup (highest probability)
  Entry 2: OB / FVG
  Entry 3: Breaker / OTE

Aturan SL Turtle Soup:
  → SL = 25% of the range
  → Jika SL terkena: kamu salah
  → Target = opposing end of range (minimum 1:3 RR)
  → Third tap+ = probabilitas tertinggi
  → Ketika benar: trade selesai sangat cepat
```

## TS Mencegah FVG Terisi

```
Turtle Soup PREVENTS Gaps from filling.
  → Ketika bullish + ada Breaker dan FVG di bawahnya:
     antisipasi FVG akan TIDAK TERISI
  → TS mempertahankan FVG sebagai unfilled liquidity pool

5 Alasan FVG tidak terisi:
  1. Breakers
  2. SMT fill (satu aset mengisi, yang lain tidak)
  3. BPRs
  4. Turtle Soups
  5. Time is up (candle sudah close)

"Turtle Soup PREVENTS Gaps from filling"
```

# Breaker — Aturan Detail

## Red Flags Breaker Tidak Valid

```
Jika breaker tidak bereaksi secara instant → bukan breaker sejati

Red Flags (breaker gagal):
  1. Consolidation tepat DI BAWAH breaker body + inside candle
  2. Close ABOVE breaker body = red flag terbesar
  3. Trading above 50% of the breaker body

Breaker Bullish rule:
  → Jika harga mencapai 50% breaker body:
     → Market tidak sebullish yang dikira
     → Atau breaker salah → rally dari PD array lain (FVG, OB)

"If the breaker doesn't give you an instant reaction, it's not a breaker."
```
